## 提示词

是指输入给 AI 模型的文本内容，用来告诉 AI 希望它做什么、怎么回答、以什么格式输出等，一个好的提示词可以节省大量的时间。

1. **传达任务**：告诉 AI 要完成什么（写代码）。
2. **设定格式**：要求输出是 vue2、vue3、使用什么工具。
3. **提供上下文**：比如 API 文档、数据结构、业务规则，参考代码。
4. **约束行为**：例如“不要解释，只输出代码”。

对于前端来说：

```text
[功能描述] + [UI/交互要求] + [数据结构] + [约束条件]
```

## 页面绘制

首选Gemini3，为现阶段最强的用户界面生成的智能体，Google 明确将 Gemini 3 定位为 **“能动态生成完整用户界面的智能体”**，其前端绘制能力能偶根据自然语言请求，实时生成可交互、可运行、样式完整的前端界面（HTML/CSS/JS 或 React/Vue 组件）。

原生多模态理解 → 精准解析 UI 意图：

- 能同时理解 文本描述 + 设计稿图像 + 原型视频。
- 示例：上传一张 截图 + 输入“用 vue3 实现这个登录页”，Gemini 3 可输出完整 vue3 代码。
- 支持从 手绘草图 或 竞品网页截图中提取布局结构。

提示语：

```
请根据我提供的设计稿图片，生成一个完整的、可运行的 [技术栈] 页面。要求如下：
1. 功能描述：[简述页面用途和核心交互]
2. UI/交互要求：[指定框架、样式方案、响应式等]
3. 内容细节：[关键文案、占位数据、图标来源]
5. 约束条件：[单文件 / 多文件 / 嵌入式 HTML]”

```

尽量使用高分辨率图片。

因为是图片，部分字体大小、间距等可能会无法准确的识别，需按照设计稿进行修改或在提示词中明确。

复杂的页面 AI关注不到太多细节，可拆分成小块让ai 单独绘制成组件。



在继续绘制页面时，可将修改的代码也附加给AI，让AI 学习设计稿的样式，参考准确的尺寸大小绘制出后续页面。
